# 认证与鉴权

确定访问以及使用网站用户的身份从来都是一个很严肃的问题，特别是在搭建具有社交性质的网站的情况下。如果访问并操作账号的用户不能被确定的话，很容易导致不可预料的后果。

所以我们需要一种兼顾用户的方便以及占用尽量低的成本的方法，来认证用户。

## 认证（Authentication）

1. 你能够收到邮箱，你就是账号的主人
2. 你知道密码，你就是账号的主人

## 应用环境

获取资源（API） ==> Token
~~确认用户身份（API的POST环境） ==> Cookie~~

### Token

采用JWT（AccessToken + RefreshToken），存放在cookie里
